# The Dude

XP said: code review is good, so pair program all the time. Testing is good, so write tests first. Integration is good, so integrate continuously.

XVC says the same thing about AI — lean into it completely. Take each XP practice and ask what it becomes when AI is fully in the loop. Not adapt it. Not water it down. Run it without the governor.

XP was limited by human execution speed. Every practice was constrained by what a human pair could actually do. AI removes those constraints. XVC isn't a reinvention of XP. It's XP ungoverned.

---

The Dude is the reference implementation of XVC on Claude Code. Like Rails to Ruby — not the thing itself, but an opinionated layer on top that makes a specific way of building practical.

At code level, The Dude is just the `~/.claude` folder, versioned and published. CLAUDE.md, commands, hooks, memory — all of it. Anyone can clone it, point it at their Claude Code, and they're running XVC. No install script, no framework to learn. Just a folder.

The Dude isn't Claude Code. It's the harness that makes XVC practices actually executable.

---

Most vibe coding is coins thrown at a slot machine — prompt, hope, get something, throw it away, try again. No discipline, no repeatability, no quality floor.

The Dude codifies what actually works so the output is consistent and principled, not lucky.

---

The Dude is a coding agent, full stop. Claude Code native. Every behavior traces back to a practice. That constraint is what makes it valuable.

In XVC, the Dude isn't a tool you prompt. It's the pair. A pair has opinions. A pair pushes back. A pair has a working style you develop together over time.
