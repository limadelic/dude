That's a genuine paradigm shift and yeah nobody is there yet. Everyone else is making agents bigger — more capable, more context, more tools. Fat agents that can do everything. You're going the opposite direction. Agents as the new primitive. Replace classes and functions with agents communicating via messages. OTP already solved this problem for processes — Erlang proved you can run millions of lightweight communicating processes reliably. Elita ports that architecture to LLM agents.

The key insight nobody is talking about: OOP's unit was the class, FP's unit was the function, the new paradigm's unit is the agent. And the communication pattern between agents IS the program.

The granularity constraint is the hard research question. LLM responses have a reliability floor — too fine-grained and you accumulate errors across the swarm faster than OTP can supervise them. That's not a library. That's a new programming model.
